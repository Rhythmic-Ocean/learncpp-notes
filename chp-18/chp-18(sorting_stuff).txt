  -selection sorting:
    -choosing the smallest element and putting it in index 0
    -choosing 2nd smalles element and putting it in index 1
    -choosing 3nd smalles element and putting it in index 2
    ....and so on until we are done.
    Impl in C++:

    template<typename T, std::size_t N>
    void sel_sort(std::array<T, N>& arr){
      int smallestIndx {};
      for(std::size_t u{}; u < N; ++u){
        smallestIndx{u}; 
        for(std::size_t v{u + 1}; v < N; ++v){
          if(arr[v] < arr[smallestIndx]{
            smallestIndx = v;
          }
        }
        std::swap(arr[u], arr[smallestIndx]);
      }
    }

  Intro to Iterators:
  -an iterator is an object designed to traverse through containers
  -different containers itself can porvide different kind of iterators
  -pointers as iterators:
        std::array arr{ 0, 1, 2, 3, 4, 5, 6 };
        auto begin{&arr[0]};
        auto end{&arr[0] + std::size(arr)}; //NOTE: Don't do &arr[std::size(arr)] cuz [] will do automatic dereferencing, and dereferencing an invalid pointer is BAD
        for(auto ptr{begin}); begin != end; ++begin){
          ...
        }
  NOTE: for noramal interations <= is preferred but for pointer iteration of a container != is preferred cuz our datatype in containers might no be able to interpret <= sign

  -iteration stuff in stl:
      std::array array{ 1, 2, 3 };
      auto begin {array.begin()};
      auto begin {array.end()};
      for (auto p{ begin }; p != end; ++p) // ++ to move to next element.
      {
          std::cout << *p << ' '; // Indirection to get value of current element.
      }
  -we also have std::end and std::begin in <iterator> library or <array> or <vector> (or other containers too? Cuz they implement stuffin iterators)
  -some operation on the iterator can invalidate the given container's iterator:
    like for vector, it could be .erase(), .push_back() etc..
      std::vector v{ 1, 2, 3, 4, 5, 6, 7 };
      auto begin {v.begin()}; -> gives back iterator
      ++begin;
      std::cout << *begin;
      v.erase(begin); -> .erase() takes in iterator and deletes it. NOw the iterator begin is invalidated
      ++begin;-> UB
    NOTE: .earse() returns the stuff next to the currently being erased stuff as iterator, so we cna do

  Timing your code:
  class Timer{
  private:
    using Clock = std::chrono::steady_clock;
    using Second = std::chrono::duration<double, std::ratio<1>>;
    
    std::chrono::time_point<Clock> m_beg {Clock::now()}

  public:
    void reset(){
      m_beg = Clock::now();
    }
    double elasped() const{
      return std::chrono::duration_cast<Second>(Clock::now() - m_beg).count();
    }
  }
  int main(){
    Timer t;
    std::cout << "Time elasped : " << t.elasped() << " seconds\n";
    return 0;
  }
