    std::array <int, 5> {};

    std::array<int, DAYS_PER_WEEK> d {}; // Using a macro (don't do this, use a constexpr variable instead)

17.4
  assigning std::array struct types:
    constexpr std::array<Houst, 9> {};
  -with CTAD:
    constexpr std::array {House{1, 2, 3}, House{1, 2, 3}, House{1, 2, 3}, House{1, 2, 3}};
  -w/o mentioning intializer type for each:
    //BUGGY!!
    constexpr std::array<House, 3> {{1, 2, 3}, {1, 2, 3}, {1, 2, 3}, {1, 2, 3}};
    NOTE: simplified std::array looks like dis:
    namespace std{
    template typename<T, std::size_t N>
    struct array{
        T imlplementation_def_name[N] {}; 
        //some other stuffs
      }
    }
    so when we do
      constexpr std::array<House, 3> {{1, 2, 3}, {1, 2, 3}, {1, 2, 3}, {1, 2, 3}};
    to the c style array inside std::array struct it looks like:
        T imlplementation_def_name[N] {1, 2, 3}, {1, 2, 3}, {1, 2, 3}, {1, 2, 3}; 
      it will try to parse {1, 2, 3} then it will see , and now we r fucked
    so the non buggy way is: use double {}, that way the struct will reach the internal array in proper format
      constexpr std::array<House, 3> {{{1, 2, 3}, {1, 2, 3}, {1, 2, 3}, {1, 2, 3}}};

  NOTE: we don't have to do double braces for everything to satisfy the internal C style array due to something called brace elison where compiler does it for us, but in some cases
        that won't happen like above and we have to do it
        so technically it might be safer to just do {{}} double braces every time!!

17.5
  Array of references:
    -we can't directly have an array of references:
      std::array<int&, 3> a {};  -> ILLEGAL!!
    -but we can do std::reference_wrapper<T> to overcome dis:
      std::array<std::reference<int>, 3> a {x, y, z}; -> now arr[0] is reference to x and so on... 
    -we can't just use operator= to a reference_wrapper type
      std::reference_wrapper<int> a = x;
      a = 3; -> ILLEGAL
      a = b; -> IS LEGAL but it's to reset the reference wrapper, basically making it reference another variable b.
    -if we wanna change value itself, we need to use .get() member functino [NOTE: DIFFERENCE FROM std::get() NON MEMBER FUNCTION!!]
      std::reference_wrapper<int> a = x;
      a.get() = 3; -> changes value of a and in turen x to 3
    -when we use it to print and stuff we don't need .get(), implicitly converted to T&:
      std::array<std::reference<int>, 3> a {x, y, z}; -> now arr[0] is reference to x and so on... 
      std::reference_wrapper<int> b = x;
      std::cout << a[0] << b;
    -using .get() is not a prob too:
      std::cout << a[0].get() << b.get();
    NOTE: to make std::reference_wrapper<T> type obj we can also do:
      int y {1};
      auto x {std::ref{y}}; -> deduced to std::reference_wrapper<int>
      OR
      int y {1};
      auto x {std::cref{y}}; -> deduced to std::reference_wrapper<const int>

      Prefer std::string_view for read-only strings (string literal symbolic constants and string parameters).

17.9
  -According to c++ standard this is UB:
    int x {};
    int* ptr {&x};
    std::cout << ptr;
    ++ptr; -> UB
    std::cout << ptr;
    it's UB cuz according to the standard, pointer arithmetic is only legal when done within the framework of an array
  -array subscripting is done via pointer arithmetic:
    *(n + ptr) -> ptr[n]; 
  - *(ptr - n) -> ptr[-n]
      we can go to previous element like this
      int a[] {1, 2, 3, 4};
      int *ptr {a[3]};
      std::cout << ptr[-1]; -> prints 3
  -we can also iterate thru loop using ponter arithmetic
    begin = arr;
    end = arr + std::size(arr);-> NOTE: ok to reference ANY pointer as long as u don't dereference it!
    for(; begin < end; ++begin){
        std::cout << begin;
    }
  -C++ foreach loop : for(int a : arr){} is implmented using similar technique:
      #include <iostream>

      int main()
      {
        constexpr int arr[]{ 9, 7, 5, 3, 1 };

        auto __begin = arr;                // arr is our begin-expr
        auto __end = arr + std::size(arr); // arr + std::size(arr) is our end-expr

        for ( ; __begin != __end; ++__begin)
        {
          auto e = *__begin;         // e is our range-declaration
          std::cout << e << ' ';     // here is our loop-statement
        }

        return 0;
      }

17.11
  -ways to initialize c style arrays:
    const char name[] { "Alex" };        // case 1: const C-style string initialized with C-style string literal
    const char* const color{ "Orange" }; // case 2: const pointer to C-style string literal
  
      In above example in case 1, it's an array, so even if "Alex" as a c style string is already in .rodata section of the program file the compiler coppies it's content into the
        stack before referencing. Leading to us having 2 copies of the same data
      For case 2, the pointer can directly reference from the .rodata section. So we will only have 1 copy so it's faster

  -when outputting a c style array/ pointers the output depends on the type of array or pointers, eg:
    int narr[]{ 9, 7, 5, 3, 1 };
    char carr[]{ "Hello!" };
    const char* ptr{ "Alex" };

    std::cout << narr << '\n'; // narr will decay to type int*  -> PRINTS THE ADDRESS OF narr
    std::cout << carr << '\n'; // carr will decay to type char* -> PRINTS Hello!
    std::cout << ptr << '\n'; // name is already type char* -> PRINTS Alex
      When u give it a char* the compiler will assume you want to content the pointer is pointing too until null termination
      However it  can have unintended consequences like:
        char c{ 'Q' };
        std::cout << &c;
        return 0;
          the above program prints Q and some garbage value until some NULL terminator is encountered.

      Better to use this if u wanna print the address of some char:
        #include <iostream>
        int main()
        {
            const char* ptr{ "Alex" };

            std::cout << ptr << '\n';                           // print ptr as C-style string
            std::cout << static_cast<const void*>(ptr) << '\n'; // print address held by ptr

            return 0;
        }

  -flattening out an array:
    template<typename T, std::size_t Row, std::size_t Col>
    using Arrar2dFlat = std::array<T, Row*Col>;

    template<typename T, std::size_t Row, std::size Col>
    class Array2dView{
    private:
      std::reference_wrapper<Array2dFlat<T, Row, Col>> m_arr{}
    public:
      Array2dView(Array2dFlat<T, Row, Col>& arr) : m_arr{arr}{}

      T& operator[](int i){return Array2dView.get()[static_cast<std::size_t>(i)];}
      const T& operator[](int i) const{return Array2dView.get()[static_cast<std::size_t>(i)];}

      //NOTE: we using () here cuz multiple paramters in [] is only supported starting c++23
      T& operator()(int i, int j){return Array2dView.get()[static_cast<std::size_t>(i * cols() + j)];}
      const T& operator()(int i, int j) const{return Array2dView.get()[static_cast<std::size_t>(i * cols() + j)];}

      //IN C++ 23
      T& operator[](int row, int col) { return m_arr.get()[static_cast<std::size_t>(row * cols() + col)]; }
      const T& operator[](int row, int col) const { return m_arr.get()[static_cast<std::size_t>(row * cols() + col)]; }

      int rows() const { return static_cast<int>(Row); }
      int cols() const { return static_cast<int>(Col); }
      int length() const { return static_cast<int>(Row * Col); }
    }
