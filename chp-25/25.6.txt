25.6 — The virtual table
  -every class with some virtual function has it
  eg:
      class Base
      {
      public:
          virtual void function1() {};
          virtual void function2() {};
      };

      class D1: public Base
      {
      public:
          void function1() override {};
      };

      class D2: public Base
      {
      public:
          void function2() override {};
      };
    here compiler wll but a pointer to vtable like this:

      class Base
      {
      public:
          VirtualTable* __vptr;
          void smthing(std::cout << "Base\n";)
          virtual void function1() {};
          virtual void function2() {};
      };

      class D1: public Base
      {
      public:
          void smthing(std::cout << "D1\n";)
          void function1() override {};
      };

      class D2: public Base
      {
      public:
          void smthing(std::cout << "D2\n";)
          void function2() override {};
      };

    for Base the vtable holds virtual function f1 and f2, and both points back to the on inside 
        Base
    for D1 class, f1() points back to D1's f1() and f2 to Base's f2()
    for D2, f1() points to Base's f1() and f2 points to D2's f2

    NOTE: since derived class are one bard base and on part the derived class itself, just the 
      pointer definition on Base is enough

    now here:
    int main(){
      D1 d {};
      Base& base {d};
      base.function1(); -> here __vptr points to D1's virtual pointer's table, and seeing that the 
        function is virtual the compiler will have the program execute thefunctin thru vtable
        instead of from the base namespace
    }
